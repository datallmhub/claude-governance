# Backend Rules — Java / Spring Boot

## Public Identifiers (IDOR Security)

- Always use `public_id UUID` (generated by `gen_random_uuid()`) in URLs and DTOs.
- Always declare `@PathVariable UUID` in controllers. Keep `Long id` internal to the persistence layer only.
- Always resolve `publicId → internal id` as the first step in the service method.

```java
@GetMapping("/{projectUid}")
public ResponseEntity<ProjectResponse> getProject(@PathVariable UUID projectUid) {
    return ResponseEntity.ok(projectService.findByPublicId(projectUid));
}
```

## Layered Architecture

- Always follow: `Controller → Service → Repository`. No layer skipping.
- Controllers validate input, call the service, return the response. No business logic.
- Services hold all business logic. No direct DB access.
- Repositories hold all query logic. No business logic.

## Naming Conventions

- Suffixes: `XxxController`, `XxxService`, `XxxServiceImpl`, `XxxRepository`, `XxxEntity`.
- Request DTOs: `CreateXxxRequest`, `UpdateXxxRequest`.
- Response DTOs: `XxxResponse`, `XxxSummaryResponse` (shortened form for lists).
- Business exceptions: `XxxNotFoundException`, `XxxAlreadyExistsException`.

## Controllers

- Annotate with `@RestController` and `@RequestMapping("/api/v1/xxx")`.
- Always use `@Valid` on `@RequestBody` parameters.
- Always return `ResponseEntity<T>` with an explicit HTTP status.
- Always map entities to response DTOs before returning. Never expose a JPA entity directly.

```java
@PostMapping
public ResponseEntity<TaskResponse> create(@Valid @RequestBody CreateTaskRequest request) {
    return ResponseEntity.status(HttpStatus.CREATED).body(taskService.create(request));
}
```

## Services

- Declare the interface in `service/`, implementation in `service/impl/`.
- Annotate the implementation with `@Service` and `@Transactional` at class level.
- Annotate read-only methods with `@Transactional(readOnly = true)`.
- Always throw a typed business exception when a resource is not found.

```java
public TaskResponse findByPublicId(UUID publicId) {
    return taskRepository.findByPublicId(publicId)
        .map(taskMapper::toResponse)
        .orElseThrow(() -> new TaskNotFoundException(publicId));
}
```

## Repositories

- Always extend `JpaRepository<T, ID>`.
- Always declare JPQL or native queries with `@Query` in the repository.
- Always filter on `organizationId` in multi-tenant queries.
- Always paginate with `Pageable` when the result set can exceed 100 rows.

```java
@Query("SELECT t FROM TaskEntity t WHERE t.projectId = :projectId AND t.organizationId = :orgId")
Page<TaskEntity> findByProject(Long projectId, Long orgId, Pageable pageable);
```

## JPA Entities

- Always use `@Entity` + `@Table(name = "snake_case_table")`.
- Always declare `@OneToMany` and `@ManyToMany` with `FetchType.LAZY`.
- Always base `equals()` and `hashCode()` on the business identifier, not the technical id.
- Always use `@CreationTimestamp` / `@UpdateTimestamp` for audit fields.

## Validation

- Always annotate DTOs with Bean Validation constraints (`@NotBlank`, `@Size`, `@Email`, etc.).
- Always create a custom `@Constraint` validator for complex business rules.
- Always use annotation-based validation; reserve manual validation for pure business logic.

## Error Handling

- Always extend `TaskFlowException(String message, ErrorCode code)` for business exceptions.
- Always handle responses in `GlobalExceptionHandler` (`@RestControllerAdvice`).
- Always return a uniform error body: `{ code, message, timestamp }`.

## Security

- Always protect endpoints with `@PreAuthorize("hasRole('USER')")` or stricter.
- Always sanitize logs: exclude JWT tokens, passwords and PII before writing.
- Always extract `organizationId` from the JWT token server-side; ignore any value sent in the request body.

## External Calls

- Always set `.responseTimeout(Duration.ofSeconds(30))` on `WebClient`.
- Always set `connectTimeout(5000)` and `readTimeout(30000)` on `RestTemplate`.
- Always wrap external calls in try/catch with structured error logging.
